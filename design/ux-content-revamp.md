# Viva - Current UX Content System

## Product language

Viva uses plain, calm language for a trust-sensitive education workflow.

- Say `assignment`, not `submission` when speaking to students.
- Say `conversation` or `Viva`, not `defense`, unless the teacher explicitly uses that term.
- Say `report` or `teacher summary`, not `dossier`, in primary UI copy.
- Say `point to revisit`, not `weakness` or `failure`.
- Say `not discussed`, not `not tested`.

## Current user journeys

### Teacher

Create a class → share invite link → see submitted work → prepare and send a Viva → review evidence → share report.

### Student

Join a class → upload an assignment → see assigned Viva → explain the work → see a shared report when the teacher releases it.

## Content principles

1. Lead with the task the person can complete now.
2. Use one primary action per screen.
3. Explain what happens next after a significant action.
4. Keep evidence, question, and answer together in reports.
5. Explain errors in plain language and give a recovery action.
6. Never use language that implies cheating, AI detection, authorship, score, or grade.
7. Keep fairness promises visible where a student needs them: content-only review, pause control, and teacher final judgement.
