# Viva - Current Visual Direction

## Visual system

Viva uses a minimalist white-and-yellow interface designed to keep attention on the current teaching task.

- **Primary yellow:** `#FBE994`
- **Page surface:** white
- **Primary text and actions:** `#171717`
- **Supporting paper surface:** `#FFF8DC`
- **Borders:** warm light neutral, `#E7E3D8`

## Layout rules

1. Start each workspace with one yellow information banner that explains the current task.
2. Use cards only for distinct decisions, evidence findings, or work areas.
3. Use whitespace before adding borders or nested cards.
4. Keep one clear primary action per screen.
5. Use black, rounded primary buttons with a hand cursor.
6. Use yellow for attention and context, never as a warning of misconduct.
7. Pair any visual state with a text label; never rely on color alone.

## Screen patterns

- **Landing:** yellow hero with clear routes to teacher, student, and judge paths.
- **Teacher workspace:** class list and selected-class work area; submitted student work is the primary decision.
- **Student workspace:** class membership, submitted work, active Vivas, and completed reports.
- **Upload:** yellow context banner above a simple class/title/file form.
- **Conversation:** readable assignment passage beside the current question and discussion context.
- **Teacher report:** yellow overview, then evidence cards that pair passage and cited conversation answer.

## Avoid

- Dense dashboards, score charts, red accusation states, or decorative status pills.
- Cards around every sentence or transcript turn.
- UI language that implies a model has made a final academic judgement.
